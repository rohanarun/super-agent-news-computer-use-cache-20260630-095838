How a computer-use cache changes repeated workflows

1. Run the workflow once

Super’s agent performs the task by actually operating a browser or desktop UI—navigation, forms, authentication, and multi-step flows.

2. Capture the action trace

Successful steps are stored as a reusable action trace rather than discarded after a single run.

3. Reuse on the next run

When the workflow repeats, Super replays and adapts the cached steps instead of reasoning from scratch.

4. Improve over time

As environments change, the cache updates—making long-lived automation more reliable.
